## ğŸ§± Tech Stack & Architecture

This solution follows a **clean, layered architecture** ensuring separation of concerns and maintainability.

| Layer | Technology | Details |
| :--- | :--- | :--- |
| **Frontend** | **Angular + Tailwind** | Component-based, Signals, Responsive Design, Reusable UI Patterns. |
| **Backend** | **Spring Boot (Java 17)** | RESTful API, Service Layer Pattern, DTO mapping. |
| **Database** | **PostgreSQL / JPA** | Relational modeling (User -> Category -> Expense), Hibernate ORM. |

### ğŸ—ï¸ Architecture Highlights
* **DTO Pattern:** strict separation between Entities and API responses.
* **Scalable Model:** Designed for multi-user SaaS scaling.
* **UX Focus:** Frontend consumes real backend APIs with loading states and error handling.
